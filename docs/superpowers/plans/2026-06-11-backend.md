# 영상 아카이브 & 검수 시스템 — Backend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the FastAPI backend for the PRD v1.1 video archive & moderation system and wire the existing `frontend/` to it.

**Architecture:** Single FastAPI app + SQLite (FTS5 keyword index + embeddings table for vector search = the PRD's "저장소 2종"). Per-video asyncio pipeline with 8 stages matching the frontend's stage names; stage failures are isolated. ffmpeg/ffprobe for media work, faster-whisper for STT fallback, Claude API (`claude-opus-4-8` default, env-configurable) for caption correction / scene analysis / frame moderation with structured outputs, Voyage AI embeddings with a local hash-embedding fallback, Telegram notify via bot HTTP API. Rules live in `backend/rules/금칙기준.md` and are sent verbatim in the moderation prompt (edit file → next video applies it).

**Tech Stack:** Python 3.14, FastAPI, uvicorn, anthropic SDK, faster-whisper (lazy), numpy, httpx, pytest (+anyio), ffmpeg/ffprobe (already on PATH).

**Stage names (must match frontend):** 자막 추출, 자막 교정, 장면 분석, 기술 검토, 프레임 샘플링, 프레임별 금칙 판정, 색인 생성, 종합 판정.

**Parallelism (PRD §5):** after metadata probe → run in parallel: [자막 추출→자막 교정], [장면 분석], [기술 검토], [프레임 샘플링→프레임별 금칙 판정]; 색인 생성 after captions+scenes; 종합 판정 last. Videos process concurrently (one asyncio task per video).
