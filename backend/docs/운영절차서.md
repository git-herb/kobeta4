# 운영 절차서 — 영상 아카이브 & 검수 시스템

## 1. 검수 흐름

```
영상 업로드 (브라우저)
    ↓
자동 분석 파이프라인 (병렬 8단계)
  ├─ 자막 추출 → 자막 교정          (임베디드 자막 없으면 음성 인식으로 생성)
  ├─ 장면 분석                      (구간별 대표 화면 AI 설명)
  ├─ 기술 검토                      (무음 / 블랙 / 프리즈 / 오디오 클리핑)
  └─ 프레임 샘플링 → 프레임별 금칙 판정
    ↓
색인 생성 (키워드 FTS + 의미 벡터)
    ↓
종합 판정 → report.json · violations.csv → 텔레그램 알림
```

- 한 단계가 실패해도 나머지 단계는 계속 진행된다. 실패 단계는 진행 화면에 오류와 함께 표시된다.

## 2. 금칙 판정 방식

- 화면을 **약 1초 간격**으로 샘플링하고, **연속 2~3장을 묶어** 동작의 흐름을 판정한다.
- 해당 구간의 **대사·소리(비명·욕설·타격음)** 를 판단 근거에 포함한다.
- 판정 기준은 `rules/금칙기준.md` 파일이 유일한 기준이다. 기준이 느슨하거나 엄격하면 이 파일만 수정한다 — 다음 영상부터 바로 반영된다.
- 프레임 처리 실패 시 해당 프레임은 `정상`(0) 처리하고 계속 진행한다.

## 3. 심각도 및 처리 기준

| 점수 | 판정 | 운영자 조치 |
|------|------|------------|
| 0 | 통과 | 없음 |
| 1–2 | 주의 | 기록만 유지 |
| 3 | 경고 (검토 필요) | **사람이 해당 프레임을 열어 최종 확인** — 자동 통과 금지 |
| 4–5 | 방영 불가 | 방영 보류, 책임자 보고 |

- 종합 판정은 프레임 중 최고 심각도를 따른다.

## 4. 리포트 확인

- 대시보드 → 영상 클릭 → **금칙 검수** 탭: 종합 판정, 타임라인(위반/검토/무음/블랙), 위반 프레임 그리드. 프레임 클릭 시 연속 3프레임·근거 규정·소리 근거 확인.
- **분석 리포트** 탭: 메타데이터, 기술 검토 구간, 장면 분석, AI 자막 교정 내역(원본 → 교정), 자막 전문.
- 파일 출력물: `data/videos/<id>/report.json`, `violations.csv` (리포트 화면에서 다운로드 가능).

## 5. 검색

- 자연어로 검색하며 모드 4종: hybrid(기본) / keyword / vector / filter.
- 각 결과는 장면 썸네일 · 타임코드 · 출처(keyword/vector/both) · 매칭 이유를 함께 표시한다.

## 6. 삭제

- 리포트 화면에서 삭제. 원본 + 리포트 + 색인 + 프레임 이미지가 모두 함께 삭제된다(복구 불가).
- 처리 중인 영상은 삭제할 수 없다.

## 7. 보안

- API 키·토큰은 `backend/.env`로만 관리한다. 코드·저장소에 키를 넣지 않는다.
