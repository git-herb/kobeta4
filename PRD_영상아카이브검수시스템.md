# PRD — 영상 아카이브 & 검수 시스템

**버전** 1.1 · **작성일** 2026-06-10 · **수정일** 2026-06-11

---

## 1. 한 줄 정의

영상을 올리면 자동으로 분석·저장하고, 자연어로 검색하며, 방영 전 금칙 검수까지 처리하는 웹 서비스

---

## 2. 목표

| # | 목표 |
|---|------|
| 1 | 영상 업로드 한 번으로 아카이브 + 검수를 동시에 처리한다 |
| 2 | 재시작 후에도 모든 데이터가 유지된다 |
| 3 | 자연어로 영상을 검색할 수 있다 |
| 4 | 모호한 장면은 자동 통과 없이 사람이 확인한다 |

---

## 3. 사용자 흐름

```
영상 업로드
    ↓
실시간 진행 상황 확인 (프로그레스 바)
    ↓
처리 완료 → 텔레그램 알림
    ↓
검색 or 검수 리포트 확인
```

---

## 4. 기능 명세

### 4-1. 업로드

- 브라우저에서 영상 파일 업로드
- 1개 이상 누적 가능
- 단계별 진행 상황을 프로그레스 바로 표시

---

### 4-2. 자동 분석 (업로드 후 자동 실행)

| 항목 | 내용 |
|------|------|
| 자막 추출 | 영상에 자막이 들어 있으면 그대로 꺼내고, **없으면 음성을 듣고 받아 적어(음성 인식) 자막을 새로 만든다** |
| 자막 교정 | AI가 오탈자·잘못 들린 단어를 고친다. **고친 부분은 리포트에서 "원본 → 교정" 형태로 따로 확인 가능** |
| 장면 분석 | 각 구간의 대표 화면을 AI가 보고 내용을 텍스트로 설명 (검색에 활용) |
| 기술 검토 | 무음 구간 / 블랙 구간 / 오디오 클리핑 / 프리즈 화면 감지 |
| 색인 생성 | 키워드 + 의미 기반 검색용 색인 저장 |

> 한 단계가 실패해도 나머지 단계는 계속 진행한다

---

### 4-3. 검색

**모드 4종**

| 모드 | 설명 |
|------|------|
| hybrid | 키워드 + 의미 점수 합산 (기본값) |
| keyword | 정확한 단어 매칭 |
| vector | 의미가 비슷한 내용 |
| filter | 날짜·카테고리 등 조건 |

**결과 표시 형식**

각 결과에 다음 세 가지를 함께 표시한다:
- **장면 이미지** (v1.1 추가) — 검색된 시점의 화면 썸네일과 타임코드를 함께 보여줘, 클릭해 보지 않아도 어떤 장면인지 바로 알 수 있다
- **출처** — keyword / vector / both 중 하나
- **이유** — 왜 이 결과가 나왔는지 한 줄 설명

---

### 4-4. 금칙 검수

**프레임 샘플링 → 판정 → 종합 리포트** 순으로 처리

**판정 방식 (v1.1 개선)**

- 영상에서 화면을 **약 1초 간격으로 촘촘하게** 뽑는다 → 뺨 때리기처럼 1~2초 만에 지나가는 장면도 놓치지 않는다
- 화면을 한 장씩 보지 않고 **연속 2~3장을 묶어서** 본다 → 멈춘 사진 한 장으로는 알기 어려운 "동작의 흐름"(때리기·밀치기·머리채 잡기 등)을 판단할 수 있다
- 그 구간의 **대사·소리(비명·욕설 등)도 판단 근거**로 함께 쓴다

**판정 카테고리** (방송심의에 관한 규정 기준, 8종)

`성표현` `폭력` `충격혐오` `유해행위` `인격권` `차별증오` `아동청소년` `광고저작권`

**심각도 등급**

| 점수 | 판정 |
|------|------|
| 0 | 통과 |
| 1–2 | 주의 |
| 3 | 경고 (검토 필요) |
| 4–5 | 방영 불가 |

**판정 원칙**

- 금칙 기준은 `rules/금칙기준.md` 파일로 관리 (코드에 포함하지 않음) — 방송심의 규정의 조문을 근거로 판정
- 기준이 느슨하거나 엄격하면 **이 파일만 고치면 다음 영상부터 바로 반영**된다 (코드 수정·재배포 불필요)
  - 예: 처음엔 "속옷 노출 = 주의(1~2)"라서 방송 중 탈의 영상이 "검토 필요"에 그쳤는데,
    기준에 "의도적 탈의·성적 어필 = 4(방영 불가)"를 명시하자 같은 영상이 "방영 불가"로 판정됨
- 모호하거나 경계 사례는 `검토필요`로 분류 → 사람이 최종 확인
- 프레임 처리 실패 시 해당 프레임은 `정상` 처리 후 계속 진행

**출력물**

| 파일 | 내용 |
|------|------|
| `report.json` | 프레임별 + 종합 판정 결과 |
| `violations.csv` | 위반 목록 |
| 위반 프레임 미리보기 | 화면에서 직접 확인 |
| `운영절차서.md` | 검수 흐름 및 판정 기준 문서 |

---

### 4-5. 알림

- 영상 처리 완료 시 텔레그램으로 결과 요약 전송

---

### 4-6. 삭제 (v1.1 추가)

- 대시보드 또는 리포트 화면에서 영상을 삭제할 수 있다
- 삭제하면 영상 원본뿐 아니라 **리포트·검색 색인·프레임 이미지 등 관련 데이터가 모두 함께 지워진다**
- 실수 방지를 위해 삭제 전 확인 창을 띄우고, 아직 처리 중인 영상은 삭제할 수 없다

---

## 5. 병렬 처리

**영상 간 병렬** — 여러 영상을 동시에 처리한다 (업로드 순서와 무관하게 동시 진행)

**영상 내 병렬** — 한 영상 안에서 아래 단계들을 동시에 실행한다

| 병렬 실행 가능 | 순서 필요 |
|--------------|----------|
| 장면 분석 | 자막 추출 → 교정 |
| 기술 검토 (무음/블랙/클리핑/프리즈) | 색인 생성 (분석 완료 후) |
| 프레임 샘플링 | 종합 판정 (프레임 판정 완료 후) |
| 프레임별 금칙 판정 | |

> 한 단계 실패가 다른 병렬 단계를 중단시키지 않는다

---

## 6. 처리 상황 & 시각화

### 단계별 진행 상황
- 각 처리 단계 이름 + 진행률(%) 실시간 표시
- 병렬로 돌아가는 단계는 동시에 각각 표시
- 실패한 단계는 오류 메시지와 함께 표시 (나머지는 계속 진행)

### 영상 타임라인
- 영상 길이를 가로 바로 표시
- 구간별 분석 결과를 색으로 구분해 타임라인 위에 오버레이
  - 예: 무음 구간(회색) / 위반 구간(빨강) / 검토필요(노랑) / 정상(초록)

### 위반 프레임 썸네일 그리드
- 위반 또는 검토필요로 분류된 프레임을 썸네일로 나열
- 각 썸네일에 타임코드 · 카테고리 · 심각도 표시
- 클릭하면 해당 프레임 상세 판정 내용 확인

### 리포트에 함께 표시되는 분석 정보 (v1.1 추가)

위반 프레임 외에 영상이 어떻게 분석됐는지도 리포트에서 한눈에 본다:

| 항목 | 내용 |
|------|------|
| 메타데이터 | 파일명 · 크기 · 길이 · 업로드 시각 · 자막을 어떻게 얻었는지 |
| 기술 검토 | 감지된 무음·블랙·프리즈·클리핑 구간 목록 |
| 장면 분석 | 분석에 사용한 화면(썸네일)과 그 설명을 구간별로 표시 |
| AI 자막 교정 내역 | 교정으로 **바뀐 문장만** "원본 → 교정"으로 표시 |
| 자막 전문 | 시간순 자막 전체 (스크롤) |

## 9. 구축 순서

| 단계 | 내용 |
|------|------|
| 1단계 | 뼈대 — 폴더 구조, 환경변수, DB 초기화, 파일 저장 경로 |
| 2단계 | 업로드 & 처리 파이프라인 — UI + 분석 + 병렬 처리 + DB 저장 |
| 3단계 | 검수 & 검색 기능 |
| 4단계 | 시각화 — 타임라인 · 썸네일 그리드 · 진행 상황 UI |
| 5단계 | 텔레그램 알림 + 운영 절차서 출력 |

## 11. 변경 이력

### v1.1 (2026-06-11)

| 변경 | 왜 바꿨나 |
|------|----------|
| 자막이 없는 영상도 음성 인식으로 자막 생성 | 자막 트랙이 없는 영상(휴대폰 촬영 등)에서 자막 추출이 항상 실패했음 |
| 금칙 판정을 "1초 간격 + 연속 2~3장 묶음" 방식으로 변경 | 10초 간격으로 한 장씩만 보던 기존 방식은 짧은 폭력 장면(뺨 때리기 등)을 통째로 놓쳤음 |
| 판정 카테고리를 방송심의 규정 기반 8종으로 교체 | 기준이 느슨해 실제 폭력 장면이 통과되는 문제가 있었음 |
| 영상·리포트 삭제 기능 추가 | 테스트 영상과 리포트를 정리할 방법이 없었음 |
| 리포트에 메타데이터·기술 검토·장면 분석·자막(교정 내역 포함) 표시 | 리포트에 위반 프레임만 있어 영상이 어떻게 분석됐는지 알 수 없었음 |
| 검색 결과에 장면 이미지(썸네일) 표시 | 글로만 나와서 어떤 장면인지 확인하려면 매번 리포트를 열어야 했음 |
| 금칙 기준에 "의도적 탈의·성적 어필 = 방영 불가(4)" 명시 | 방송 중 옷을 벗는 영상이 "속옷 노출 = 경미" 기준에 묶여 "검토 필요"에서 멈췄음 |
